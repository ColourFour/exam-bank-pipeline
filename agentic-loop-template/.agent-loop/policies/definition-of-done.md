# Definition of Done

An iteration is done only when:

1. The planned acceptance criteria are satisfied.
2. New or changed tests/checks verify the behavior.
3. Existing tests still pass or known failures are explicitly justified.
4. The repo has no unnecessary generated artifacts.
5. The implementation report lists every changed file and why it changed.
6. The auditor either passes the iteration or identifies concrete follow-up fixes.
7. No new dependency, script, route, state model, data format, or abstraction was added without justification.
